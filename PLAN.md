# World Model — Implementation Plan

**Project:** Playable Genie-style video world model + an agent trained *inside* it, evaluated back in the real game.
**Domain:** CoinRun (OpenAI Procgen) for the build; a fresh Stable-Retro title for the showcase (see *Two-game staging*).
**Compute:** Multi-node GPU cluster.

---

## The one-sentence pitch

> I learn a controllable world model from raw CoinRun video (no action labels), train an agent entirely inside that learned model, and show its policy transfers back to the real game.

That headline — real-game score from a policy that never trained on the real game — is the whole point. Everything below builds to it.

---

## Reading list (in order)

1. **World Models** — Ha & Schmidhuber, 2018. The origin; VAE + RNN + controller.
2. **DreamerV3** — Hafner et al., 2023. RSSM, symlog, two-hot rewards, KL balancing, learning in imagination. This is the agent-training half.
3. **Genie** — Bruce et al., 2024. Tokenizer + unsupervised latent action model + dynamics. The world-model half.
4. **DIAMOND** — Alonso et al., 2024. Diffusion dynamics (your ablation comparison).
5. **VQ-VAE** (van den Oord, 2017) + **FSQ** (Mentzer et al., 2023). Tokenizer; use FSQ.
6. **MaskGIT** — Chang et al., 2022. Parallel token decoding for interactive framerate.
7. *Optional:* **IRIS** (Micheli et al., 2023) — transformer world model + agent on Atari, closest prior art to the full loop.

## Why CoinRun

2D platformer (Genie's proven domain), procedurally generated so training video is unlimited and cheap, tiny discrete action space (left / right / jump) that makes latent-action verification easy, and it renders fast for both data collection and the live loop. Fallback to **Sokoban** if visual variety fights you.

## Two-game staging

Build and debug the entire pipeline on CoinRun; when the full loop works, swap the **game, not the code**, for the headline result. Only `data/collect.py` and `eval/real_game_eval.py` touch the environment, so the swap is cheap. Every failure during the build happens on a domain where it's diagnosable (unlimited procedural data, 3 actions, known baselines) — never debug a new pipeline on a new domain.

The showcase domain is a **Stable-Retro title no world-model paper has used** — candidates: *Kirby's Adventure*, *Gradius*, *Castlevania* (NES); *Airstriker-Genesis* is the legally-cleanest fallback (free homebrew ROM bundled with stable-retro). Stable-Retro provides RAM-based reward functions for the real-game evaluation. Nearest prior art on Procgen is LAPO (latent-action policies from video, ICLR 2024), so a fresh showcase domain also differentiates the final result.

## Build stages — each is its own entrypoint and checkpoint

Never debug stage N by retraining stage N-1. Get one vertical slice working on a tiny data subset before scaling to the cluster.

### Stage 0 — Data pipeline
- Run a random + lightly-scripted policy in CoinRun; save frames (and, for later evaluation only, the true actions/rewards — the world model must NOT see actions).
- Target a few hundred hours of frames; start with a tiny subset to debug.
- **Gate:** dataloader streams frame sequences at full GPU utilization.

### Stage 1 — Tokenizer
- FSQ autoencoder on individual frames.
- **Gate:** reconstructions are sharp. Do not proceed on blurry output — everything downstream depends on this.

### Stage 2 — Latent action model (the Genie trick)
- Given (x_t, x_{t+1}): a small encoder outputs a *discrete* latent action z; a decoder reconstructs x_{t+1} from x_t + z. Tiny codebook (start 6–8 codes) forces z to capture "what changed."
- Fully unsupervised — no action labels.
- **Gate / key figure:** fix x_t, sweep z, confirm each code produces a consistent, distinguishable movement. Save this sweep as a figure — it's your best portfolio visual and proves controllability.

### Stage 3 — Dynamics model
- Autoregressive transformer over tokens, conditioned on past tokens + latent action, MaskGIT-style decoding. (Start here; it's simpler than diffusion.)
- Port DreamerV3 stability tricks (symlog, normalization, KL balancing).
- **Gate:** multi-step rollouts stay coherent for tens of frames.

### Stage 4 — Interactive loop
- Map keyboard keys → the latent action codes discovered in Stage 2 → dynamics rollout → live render.
- **Gate:** you can "play" the model. Record GIFs.

### Stage 5 — Agent trained INSIDE the world model  ← the headline
- Add a reward head to the world model (predict CoinRun reward from latent state; trained on the held-out true rewards, or a shaped proxy like progress).
- Train a **DreamerV3-style actor-critic purely in imagination** — the agent only ever sees rollouts generated by your learned model, never the real game.
- The agent acts via the *learned latent action codes*, so map those codes back to real CoinRun controls using the Stage-2 correspondence.
- **Evaluation (the money result):** deploy the imagination-trained policy in the *real* CoinRun and measure score / completion rate. Compare against:
  - a random policy (floor),
  - a PPO agent trained on the real game with matched environment interactions (reference),
  - ablation: agent trained inside a *worse* world model (e.g. fewer dynamics params) to show model quality drives agent quality.
- **Gate:** the imagination-trained agent beats random in the real game. Anything above that is a strong result.

### Stage 6 — The research ablation
- Swap transformer dynamics → DIAMOND-style diffusion dynamics. Compare on FVD, long-horizon drift, and downstream agent score.
- One clean axis, honest numbers, confidence intervals over ≥3 seeds.

### Stage 7 — Domain swap (the showcase)
- Re-run Stages 0–5 on the chosen Stable-Retro title: new collection wrapper + RAM-based reward for evaluation; everything else unchanged.
- Known risks vs CoinRun: one game's worth of levels (less visual variety), scrolling/parallax strains the tokenizer, and ~8–12 effective button combos strain the tiny latent-action codebook — grow the codebook before growing the model.
- **Gate:** the full loop — playable model + imagination-trained agent beating random — reproduced on the new game.

## Metrics

- **World model:** FVD (rollout quality), controllability probe (does action k have a consistent effect?), long-horizon drift (FVD vs rollout length).
- **Agent (headline):** real-game CoinRun score / completion rate for the imagination-trained policy, vs random and vs real-env PPO.
- **Coupling result:** agent score as a function of world-model quality — the plot that ties the whole project together.

## De-risking

- Tokenizer blurry → FSQ over VQ; drop resolution/frame count before dropping ambition.
- Latent actions meaningless → shrink codebook, increase the bottleneck.
- Agent won't learn in imagination → check reward-head accuracy first; a wrong reward model dooms the agent. Verify imagined rollouts match real dynamics before blaming the RL.
- Whole thing too big → get the full loop working on a *single CoinRun level* first, then scale to procedural generation.

**Rule:** one working vertical slice on a tiny domain beats an ambitious broken pipeline in every portfolio review.
